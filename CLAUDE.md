# 量化交易理论研究系列 - 写作规范

## 一、核心原则

### 1. 可读性与完整性的平衡

文章应有充分的内容让读者能够认识和深刻理解主题，又不至于内容多到难以阅读和理解。

### 2. 理论与研究驱动

每篇文章必须有坚实的理论和研究基础：

- **学术支撑**：引用相关学术论文，说明理论来源和实证证据
- **数据支持**：提供具体的回测数据、统计指标（如夏普比率、最大回撤）
- **多方观点**：既展示支持证据，也要呈现反对观点和局限性

**示例**：
```markdown
**学术支持**：Moskowitz, Ooi & Pedersen (2012) 在 58 种期货合约上
发现显著的时间序列动量效应，年化 Alpha 约 8%。

**反对观点**：部分研究（如 Grobys 2019）在加密货币市场未发现
显著的动量收益，表明市场效率在提高。
```

### 3. 覆盖加密货币市场

**策略类文章**（`strategies/` 目录）应包含加密货币相关内容。理论基础类文章（`core/` 目录）可根据相关性决定是否包含。

策略文章应覆盖：

- **市场特征**：高波动性、24/7 交易、流动性分散等独特特点
- **策略适配**：参数调整建议（如更短的回望期）
- **实证数据**：加密货币市场的策略表现和风险特征
- **特有工具**：资金费率、减半周期、链上指标等

**应包含的内容**：
| 维度 | 示例内容 |
|-----|---------|
| 市场特征 | 波动率对比、流动性结构、交易时段特点 |
| 策略表现 | 与传统资产对比的回测结果 |
| 参数适配 | 推荐的参数范围及原因 |
| 风险因素 | 市场效率变化、交易成本、流动性风险 |
| 学术研究 | 相关论文引用 |

## 二、术语定义规则

### 1. 核心概念：文章开篇定义

如果某个概念是文章的核心主题，应在正文开始前专门定义。

**格式**：
```markdown
## 核心概念定义

### 什么是 XXX？

**XXX** 是指...

**举例**：...
```

### 2. 重要术语：首次使用时定义

术语在首次出现的位置进行定义，而非统一放在开头或结尾。

**定义方式**（根据术语重要程度选择）：

| 重要程度 | 定义方式 | 示例 |
|---------|---------|------|
| 高 | 独立段落 | 套利、做空 |
| 中 | 引用块 | P/B、P/E、夏普比率 |
| 低 | 括号内解释 | 随机游走 |

**示例**：

```markdown
<!-- 高重要度：独立段落 -->
**什么是套利？**

**套利 (Arbitrage)** 是指利用价格差异获取无风险利润的行为。

<!-- 中重要度：引用块 -->
> **P/B (Price-to-Book)** = 股价 / 每股净资产，衡量市场对公司资产的估值

<!-- 低重要度：括号解释 -->
未来价格变动不可预测（随机游走，即价格变化像掷骰子一样随机）
```

### 3. 避免重复定义

同一术语只在首次出现时定义，后续使用不再重复。如果需要在不同章节引用，可写"如前所述"或直接使用。

## 三、解释与例子规则

### 1. 每个重要概念必须有例子

抽象概念必须配合具体例子，帮助读者理解。

**反面示例**（只有抽象描述）：
```markdown
**框架效应 (Framing Effect)**

相同信息的不同表述方式会导致不同决策。
```

**正面示例**（有具体例子）：
```markdown
**框架效应 (Framing Effect)**

相同信息的不同表述方式会导致不同决策。

**经典例子**：
- "这只股票有 70% 的概率盈利" vs "这只股票有 30% 的概率亏损"
  ——同样的信息，但前者让人更愿意买入
```

### 2. 表格不能孤立存在

表格前后必须有说明文字，解释表格的含义和用途。表格后如有必要应补充例子。

**反面示例**：
```markdown
**3. 实施成本**

| 成本类型 | 说明 | 典型水平 |
|---------|------|---------|
| 交易佣金 | 经纪商费用 | 0.01%-0.1% |
| 买卖价差 | 流动性成本 | 0.02%-2% |
```

**正面示例**：
```markdown
**3. 实施成本**

套利不是免费的，每一笔交易都有成本。当这些成本加起来超过预期收益时，套利就变得不划算。

| 成本类型 | 说明 | 典型水平 |
|---------|------|---------|
| 交易佣金 | 经纪商费用 | 0.01%-0.1% |
| 买卖价差 | 流动性成本 | 0.02%-2% |

**例子**：你发现一只小盘股被低估 5%。但买卖价差是 1%，买入时市场冲击 1%，卖出时再各付一次。总成本约 4%，几乎吃掉全部利润。
```

### 3. 复杂概念需要通俗解释

学术性的对比表格或公式后，应补充通俗易懂的解释。

**示例**：
```markdown
| 维度 | 期望效用理论 | 前景理论 |
|-----|------------|---------|
| 效用载体 | 最终财富水平 | 相对参考点的变化 |
| 损失态度 | 对称处理 | 损失厌恶 |

**通俗解释**：
- **效用载体**：传统理论认为你在乎的是"总共有多少钱"，前景理论认为你在乎的是"比之前多了还是少了"
- **损失态度**：传统理论认为赚100和亏100的心理感受对称，前景理论发现亏100的痛苦远大于赚100的快乐
```

## 四、例子的写作标准

### 1. 例子要具体，不要抽象

**反面示例**：
```markdown
模型风险是指你的模型可能是错的。
```

**正面示例**：
```markdown
**价值陷阱 (Value Trap)** 例子：
- 你看到：P/B = 0.5，按模型"被低估 50%"
- 你的行动：大量买入
- 实际情况：公司主营业务正在被淘汰，资产即将减值
- 结果：股价继续下跌，你亏损
```

### 2. 例子要有完整的逻辑链

好的例子应包含：背景 → 行动 → 结果 → 教训

**示例**：
```markdown
**例子**：你管理一只基金，发现市场处于泡沫中。理性的做法是做空或离场。但如果泡沫继续膨胀一年，你的业绩将大幅落后同行，客户会赎回，你可能被解雇。所以很多基金经理选择"跟随泡沫"——即使知道它终将破裂。
```

### 3. 例子要贴近读者

优先使用：
- 中国市场的例子（如沪深300指数）
- 日常生活类比（如彩票、飞机恐惧）
- 具体数字（如"5%的错误定价"而非"小幅错误定价"）

## 五、结构规则

### 1. 避免内容重复

不同章节讨论同一概念时，只在首次出现处完整定义，后续章节聚焦于新的角度。

**示例**：市场有效性的三种形式在"核心概念定义"中完整介绍后，在"视角A"中只展示实证结论，不重复定义。

### 2. 保持层次清晰

```
# 文章标题
## 大章节（核心概念定义、理论视角、常见陷阱等）
### 中章节（视角A、视角B等）
#### 小章节（具体理论、案例等）
**加粗标题**（最小单元，如"1. 损失厌恶"）
```

### 3. 使用过渡句

章节之间使用过渡句，帮助读者理解逻辑关系。

**示例**：
```markdown
**局限**：行为金融解释了为什么会有错误定价，但没有解释为什么套利者不能消除它。

这引出了下一个视角。

---

### 视角 C：限制套利
```

## 六、公式与统计术语规则

### 1. 公式必须解释变量含义

```markdown
$$\alpha = R_p - R_{expected}$$

其中：
- $R_p$ = 投资组合的实际收益
- $R_{expected}$ = 根据风险水平应该获得的收益
```

### 2. 统计术语要解释含义

**反面示例**：
```markdown
传统 t > 2 标准不够，应要求 t > 3
```

**正面示例**：
```markdown
传统 t > 2 标准不够，应要求 t > 3

> **t 统计量**衡量的是"结果是偶然出现的可能性有多大"。t > 2 意味着偶然概率小于 5%；t > 3 意味着偶然概率小于 0.3%。
```

## 七、篇幅与内容深度

### 1. 内容优先于篇幅

- **不设行数上限**：以内容完整性为首要目标
- **深度优于简洁**：宁可详尽也不遗漏重要内容
- **必要时可拆分**：如果单篇确实过长且可独立成章，考虑拆分

### 2. 必须包含的内容

每篇策略文章应覆盖：
- 核心概念定义与历史背景
- 理论基础与学术支持
- 策略机制与实施方法
- 传统资产市场的应用与表现
- **加密货币市场的应用与适配**
- 风险、局限与失败场景
- 相关策略的关联与对比

### 3. 精简原则（在保证深度的前提下）

- 保留关键公式，删除冗长推导过程
- 保留代表性案例，删除重复性案例
- 删除与主题关系不紧密的延伸内容

### 4. 代码处理

- 核心系列文章：不包含代码，只保留关键公式
- 如需代码示例，放在独立的"实践篇"或附录中
